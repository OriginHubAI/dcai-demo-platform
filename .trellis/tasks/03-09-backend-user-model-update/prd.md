## Goal
Update Django `user` app models to match the `pg_schema_v1_references` and production standards.

## Requirements

- [x] Change `db_table` to `my_user`.
- [x] Add missing fields:

    - `updated_at` (DateTimeField, auto_now=True)

### 2. Update `InviteCode` Model
- [x] Change `db_table` to `invite_code`.

- [x] Add `updated_at` (DateTimeField, auto_now=True).
- [x] Ensure `id` is compatible with SQL (currently UUID, SQL uses integer generated by default). 
    - *Decision*: Keep UUID for Django's primary key as it's more flexible for distributed systems, but ensure SQL `id` in reference is understood as primary key.

### 4. Cleanup/Sync `VerificationCode`
- [x] Determine if unified `VerificationCode` should be replaced or kept alongside specialized ones.
    - *Decision*: Keep `VerificationCode` for generic purposes but add `EmailVerificationCode` for specific high-performance email-only verification as per schema.
